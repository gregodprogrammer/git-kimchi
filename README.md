# Git Kimchi — Beginner's Guide to Git & GitHub for DevOps

A comprehensive, hands-on guide taking you from Git novice to confident contributor. Built for DevOps students who learn by doing.

## What You'll Learn

This multi-file documentation guide covers everything from basic Git commands to advanced DevOps workflows including:

- **Core Git**: init, config, staging, commits, history, diff, undo
- **GitHub Collaboration**: repos, forks, pull requests, issues, code review
- **Branching Strategies**: GitFlow, trunk-based development
- **DevOps Integration**: Git hooks, GitHub Pages, self-hosted Git (GitLab/Gitea)

## Structure

Each section includes:
- Copy-pasteable shell commands with expected terminal output
- Hands-on exercises with solutions
- Progressive difficulty — follow linearly from beginner to advanced

## Getting Started

**Prerequisites:**
- Terminal/shell access (bash, zsh, PowerShell)
- Git installed (`git --version` should work)
- GitHub account (free tier is fine)

**Start here:** [00-index.md](./00-index.md) — the complete table of contents and reading guide.

## Quick Check

Verify your setup:

```bash
# Check Git is installed
git --version
# Expected: git version 2.34.0 (or newer)

# Check GitHub CLI (optional but recommended)
gh --version
```
